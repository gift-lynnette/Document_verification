# Chapter Four Screenshot Guide

Use screenshots in Chapter Four only where they support a major function, workflow step, or result. Do not screenshot every page. For this system, the strongest screenshot points are:

1. `Login page`
Purpose: Shows the system entry point and authentication interface.
Suggested caption: `Figure 4.1 Login interface for the KIU automated verification system.`

2. `Student registration page`
Purpose: Supports the student onboarding part of the implementation.
Suggested caption: `Figure 4.2 Student registration page used to create user accounts.`

3. `Student dashboard`
Purpose: Shows status tracking, notifications, and access to green card functions.
Suggested caption: `Figure 4.3 Student dashboard showing submission tracking and quick actions.`

4. `Document submission page`
Purpose: Important because it proves the system collects academic and payment documents.
Suggested caption: `Figure 4.4 Document submission interface for uploading student verification records.`

5. `Admissions dashboard or admissions review page`
Purpose: Shows how admissions staff review, approve, reject, or request resubmission.
Suggested caption: `Figure 4.5 Admissions verification interface for document review and registration approval.`

6. `Finance dashboard or payment verification page`
Purpose: Shows the finance clearance stage of the workflow.
Suggested caption: `Figure 4.6 Finance verification interface for confirming payment and clearance decisions.`

7. `Admin dashboard`
Purpose: Shows user management, system monitoring, and institutional oversight.
Suggested caption: `Figure 4.7 Administrative dashboard for system monitoring and management.`

8. `Generated green card or green card download view`
Purpose: This is one of the most important result screenshots because it is the final system output.
Suggested caption: `Figure 4.8 Generated digital green card produced by the system.`

9. `Public verification page`
Purpose: Shows that the issued card can be validated externally.
Suggested caption: `Figure 4.9 Public verification page for confirming green card authenticity.`

## Best Placement in Your Report

- `System Development`
Use login, registration, student dashboard, document submission, admissions dashboard, finance dashboard, and admin dashboard screenshots.

- `System Testing / Validation`
Use the public verification page and generated green card screenshot as evidence that the end-to-end workflow works.

- `Results`
Use the green card screenshot and public verification screenshot again if needed, because they are the clearest proof of the final system output.

## Recommended Number

For Chapter Four, `6 to 9 screenshots` is enough. A good balanced set is:

- Login page
- Registration page
- Student dashboard
- Document submission page
- Admissions review page
- Finance review page
- Admin dashboard
- Generated green card
- Public verification page

## Important Notes

- Make sure every screenshot has a figure number and caption.
- Mention each screenshot in the paragraph before or after it.
- Use clear screenshots with visible page titles.
- Avoid screenshots with too much blank space.
- If possible, crop only the most relevant part of the page.
